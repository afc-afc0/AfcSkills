The user has pasted a **carry-over packet** — compact notes from another session. Get oriented so you can pick up the thread together.

1. **Read the packet** — goal, state, next steps, flagged notes, file list.
2. **Read the actual code** it points to. The packet references files/PRs by path instead of pasting them — open them and ground yourself in what's really there.
3. **Explain it back** in plain English: what the work is, how the relevant pieces actually work, where things stand, and what's next. Flag anything that doesn't line up with the code.

Then **stop** — this is a starting point for discussion, not a work order. Ask where the user wants to begin; don't make changes until they say so.
